Define a metric

Creating metrics

1. What you going to use the metric for?
A. Invariant checkings
  • used for data sanity checking
  • metrics that should not change from experiment to control group
  • e.g.: check if populations has the same distribution
B. Evaluation metrcis B.1) High-level bussines metrics
  • what the market share is?
  • how much revenue you make?
  • how much users you have?
B.2) Detailed metrics
  • on phenomenon measurable characteristics
  • on user experience for example: how long they stop on the page
Difficult metrics
  • don't have direct access to the data
  • takes to much time
  • techniques to approach difficult metrics
2. Making a definition
1. High-level concept
  • come up with a high-level concept for the metric
  • every word should be understand
  • the business goal is a possible start point
  • e.g. active users, or click-through-probability
  • additional techniques for brainstorming & validating metrics
  • kinds of metrics:
Count
  • number of items that matches a specified criteria
Ratios
  • \(\div\) of counts
  • measure usability
  • use if interested on increase rate of progression
Probabilities
  • unique events considered
  • measure progression
2. Specify details
  • detailed definition of all concepts
  • determine word-by-word which ones needs explanation
3. Summarize
  • summarize all definition in a single metric
  • sum, count, average, median
4*. Composite metric
  • if have multiple metrics → can create composite metric
  • OEC: overall evaluation criterion
